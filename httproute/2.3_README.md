### 2.2 HTTP 리다이렉트 및 리라이트

HTTPRoute 리소스는 필터를 사용하여 클라이언트에 리다이렉트를 발행하거나 업스트림으로 전송된 경로를 재작성할 수 있습니다.

**참고**: Redirect 및 Rewrite 필터는 상호 배타적입니다. 규칙은 두 필터 유형을 동시에 사용할 수 없습니다.

### Redirect

리다이렉트는 클라이언트에 HTTP 3XX 응답을 반환하여 다른 리소스를 검색하도록 지시합니다.

##### 지원되는 상태 코드

Gateway API는 다음 HTTP 리다이렉트 상태 코드를 지원합니다:

- **301 (Moved Permanently)**: 리소스가 영구적으로 새 위치로 이동했음을 나타냅니다. HTTP에서 HTTPS로의 영구 업그레이드 또는 영구 URL 변경에 사용합니다.
- **302 (Found)**: 리소스가 일시적으로 다른 위치에서 사용 가능함을 나타냅니다. 상태 코드를 지정하지 않으면 기본값입니다.
- **303 (See Other)**: 요청에 대한 응답을 GET 메서드를 사용하여 다른 URL에서 찾을 수 있음을 나타냅니다. POST 요청 후 확인 페이지로 리다이렉트하는 데 일반적으로 사용됩니다.
- **307 (Temporary Redirect)**: 302와 유사하지만 리다이렉트를 따를 때 HTTP 메서드가 변경되지 않음을 보장합니다.
- **308 (Permanent Redirect)**: 301과 유사하지만 리다이렉트를 따를 때 HTTP 메서드가 변경되지 않음을 보장합니다.

#### 경로 리다이렉트(method preserve redirect)

경로 리다이렉트는 HTTP Path Modifier를 사용하여 전체 경로 또는 경로 접두사를 교체합니다.

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: redirect-gateway
  namespace: web
spec:
  addresses:
  - type: "IPAddress"
    value: 192.168.48.202
  gatewayClassName: f5-gateway-class
  listeners:
  - name: http
    protocol: HTTP
    port: 80
    allowedRoutes:
      namespaces:
        from: "All"
      kinds:
      - kind: HTTPRoute
  - name: https
    protocol: HTTPS
    port: 443
    allowedRoutes:
      namespaces:
        from: "All"
      kinds:
      - kind: HTTPRoute
    tls:
      mode: Terminate
      certificateRefs:
      - name: local-test-tls-cert
        namespace: sec-infra
        kind: Secret
        group: ""
```

HTTPRoute를 설정합니다. Gateway API Spec으로는 filter의 RequestRedirect type으로 기능이 제공하지만 현재 F5 Gateway API는 iRule 기반으로 제공합니다. 
```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: method-preserving-redirect
  namespace: web
spec:
  parentRefs:
  - name: redirect-gateway
    sectionName: http
  hostnames:
  - www.f5bnk.com
  rules:
  - matches:
    - path:
        type: PathPrefix
        value: /coffee
    filters:
    - type: RequestRedirect
      requestRedirect:
        path:
          type: ReplaceFullPath
          replaceFullPath: /tea
        statusCode: 301 
  - backendRefs:
    - name: coffee-svc
      port: 80
```

해당 기능은 Gateway API Spec으로 지원하지 않고 iRule 기반으로 제공하기 때문에 아래와 같은 iRule의 설정이 필요합니다. 
```yaml
apiVersion: "k8s.f5net.com/v1"
kind: F5BigCneIrule
metadata:
  name: irule-method-preserving-redirect
  namespace: web
spec:
  iRule: >
    when HTTP_REQUEST {
    if { [string tolower [HTTP::uri]] starts_with "/coffee" } {
        set new_uri [string map -nocase {"/coffee" "/tea"} [HTTP::uri]]
        HTTP::respond 307 Location "http://[HTTP::host]$new_uri" }
    }
```

iRule을 F5 Gateway에 설정하기 위한 BNKNetPolicy를 생성하여, iRule을 지정된 Gateway의 `sectionName`에 연결합니다.
```yaml
apiVersion: "gateway.k8s.f5net.com/v1alpha1"
kind: BNKNetPolicy
metadata:
  name: netpolicy-method-preserving-redirect
  namespace: web
spec:
  extensionRefs:
  - group: k8s.f5net.com
    kind: F5BigCneIrule
    name: irule-method-preserving-redirect
  targetRefs:
  - group: gateway.networking.k8s.io
    kind: Gateway
    name: redirect-gateway
    sectionName: "http"
```

`http://www.f5bnk.com/coffee`에 대한 요청은 모두 `location: http://www.f5bnk.com/tea`가 포함된 리다이렉트를 받습니다.
