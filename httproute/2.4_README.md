#### POST Method를 303(See Other) Redirect를 사용해서 GET 요청으로 변환 및 전송

HTTP POST 트래픽을 303 See Other Redirect를 사용하여 GET 요청으로 변환할 수 있습니다. HTTP 80 서비스 기반의 Gateway를 생성합니다.

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: post-redirect-get-gw
  namespace: web
spec:
  addresses:
  - type: "IPAddress"
    value: 192.168.48.202
  gatewayClassName: f5-gateway-class
  listeners:
  - name: http-post-redirect-get
    protocol: HTTP
    port: 80
    allowedRoutes:
      namespaces:
        from: "All"
      kinds:
      - kind: HTTPRoute
```

POST Method을 사용하는 특정 URL에 대해서 GET 요청으로 변환하는 HTTP Route를 생성합니다.

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: http-post-redirect-get
  namespace: web
spec:
  parentRefs:
  - name: post-redirect-get-gw
    sectionName: http-post-redirect-get
  hostnames:
  - www.f5bnk.com
  rules:
  - matches:
    - path:
        type: Exact
        value: /latte/coffee
      method: POST
    filters:
    - type: RequestRedirect
      requestRedirect:
        path:
          type: ReplaceFullPath
          replaceFullPath: /black/tea
  - backendRefs:
    - name: httpbin
      port: 8080
```

F5 BNK 2.1 버전에서는 HTTP Route의 303 Redirect 기능을 직접 지원하지 않기 때문에 이를 지원하기 위한 iRule을 생성합니다.

```yaml
apiVersion: "k8s.f5net.com/v1"
kind: F5BigCneIrule
metadata:
  name: irule-post-redirect-get
  namespace: web
spec:
  iRule: >
    when HTTP_REQUEST {
      if { [HTTP::method] eq "POST" } {
      HTTP::respond 303 Location "http://[getfield [HTTP::host] ":" 1]/black/tea"
      }
    }
```

생성한 iRule을 Gateway에 적용합니다.

```yaml
apiVersion: "gateway.k8s.f5net.com/v1alpha1"
kind: BNKNetPolicy
metadata:
  name: netpolicy-post-redirect-get
  namespace: web
spec:
  extensionRefs:
  - kind: F5BigCneIrule
    name: irule-post-redirect-get
    group: k8s.f5net.com
  targetRefs:
  - kind: Gateway
    name: post-redirect-get-gw
    group: gateway.networking.k8s.io
    sectionName: http-post-redirect-get
```
