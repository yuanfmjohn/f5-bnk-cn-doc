### HTTP 헤더 수정

HTTP 헤더 수정은 들어오는 요청의 HTTP 헤더를 추가, 제거 또는 수정하는 프로세스입니다. 

** HTTP 요청 헤더의 추가 **

Gateway 설정:

```yaml
fapiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: modifier-gw
  namespace: web
spec:
  addresses:
  - type: "IPAddress"
    value: 192.168.48.202
  gatewayClassName: f5-gateway-class
  listeners:
  - name: http-modifier
    protocol: HTTP
    port: 80
    allowedRoutes:
      namespaces:
        from: "All"
      kinds:
      - kind: HTTPRoute
```

HTTP Route 설정

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: http-modifier
  namespace: web
spec:
  parentRefs:
  - name: modifier-gw
    sectionName: http-modifier
  rules:
  - matches:
    - path:
        type: PathPrefix
        value: /goods
    filters:
    - type: RequestHeaderModifier
      requestHeaderModifier:
        add:
        - name: Host
          value: new.f5bnk.com
  - matches:
    - path:
        type: PathPrefix
        value: /coffee
    filters:
    - type: RequestHeaderModifier
      requestHeaderModifier:
        set:
        - name: User-Agent
          value: Modified-Chrome
  - matches:
    - path:
        type: PathPrefix
        value: /tea
    filters:
    - type: RequestHeaderModifier
      requestHeaderModifier:
        remove: ["X-tea-id"]
  - filters:
    - type: ResponseHeaderModifier
      responseHeaderModifier:
        add:
        - name: X-XSS-Protection
          value: "1; mode=block"
  - filters:
    - type: ResponseHeaderModifier
      responseHeaderModifier:
        set:
        - name: X-Frame-Options
          value: DIFFERENT
  - filters:
    - type: ResponseHeaderModifier
      responseHeaderModifier:
        remove: ["X-Content-Type-Options"]
  - backendRefs:
    - name: httpbin
      port: 8080
```

위 HTTP Route 설정에서 각각의 filter type별 동작은 아래와 같습니다.

**헤더 추가:**
요청이 조건에 매치될 경우 설정된 HTTP 요청 헤더를 삽입하여 전달합니다.
```yaml
- matches:
    - path:
        type: PathPrefix
        value: /goods
    filters:
    - type: RequestHeaderModifier
      requestHeaderModifier:
        add:
        - name: Host
          value: new.f5bnk.com
```

**헤더 수정:**
요청이 조건에 매치될 경우 설정된 HTTP 요청 헤더를 수정하여 전달합니다.
```yaml
- matches:
    - path:
        type: PathPrefix
        value: /coffee
    filters:
    - type: RequestHeaderModifier
      requestHeaderModifier:
        set:
        - name: User-Agent
          value: Modified-Chrome
```

**헤더 제거:**
요청이 조건에 매치될 경우 설정된 HTTP 요청 헤더를 삭제하고 전달합니다.
```yaml
- matches:
    - path:
        type: PathPrefix
        value: /tea
    filters:
    - type: RequestHeaderModifier
      requestHeaderModifier:
        remove: ["X-tea-id"]
```

#### 응답 헤더 수정

응답 헤더 수정은 요청 헤더 수정과 매우 유사한 구문을 사용하지만 다른 필터(`ResponseHeaderModifier`)를 사용합니다.

**여러 헤더 추가:**
HTTP 요청이 조건에 매치되면, 아래에 설정된 응답 헤더를 추가, 삭제 또는 수정할 수 있습니다. 
```yaml
- filters:
    - type: ResponseHeaderModifier
      responseHeaderModifier:
        add:
        - name: X-XSS-Protection
          value: "1; mode=block"
  - filters:
    - type: ResponseHeaderModifier
      responseHeaderModifier:
        set:
        - name: X-Frame-Options
          value: DIFFERENT
  - filters:
    - type: ResponseHeaderModifier
      responseHeaderModifier:
        remove: ["X-Content-Type-Options"]
```

F5 Gateway API Spec은 현재 Native 방식으로 요청 및 응답 헤더의 삽입, 삭제 또는 수정을 제공하지 않기 때문에 이를 지원하기 위한 iRule을 설정합니다.

```yaml
apiVersion: "k8s.f5net.com/v1"
kind: F5BigCneIrule
metadata:
  name: irule-modifier
  namespace: web
spec:
  iRule: >
    when HTTP_REQUEST {
      if { [HTTP::uri] starts_with "/headers" } {
         HTTP::header replace "User-Agent" "Chrome"
      } elseif { [HTTP::uri] starts_with "/coffee" } {
                 HTTP::header remove "X-tea-id"
        } else {
               HTTP::header insert "Host" "new.f5bnk.com"
          }
    }

    when HTTP_RESPONSE {
      if { [HTTP::header exists "X-Frame-Options"] } {
         HTTP::header replace "X-Frame-Options" "DIFFERENT"
      }
      if { [HTTP::header exists "X-Content-Type-Options"] } {
         HTTP::header remove "X-Content-Type-Options"
      }
      if { !([HTTP::header exists "X-XSS-Protection"]) } {
         HTTP::header insert "X-XSS-Protection" "1; mode=block"
          }
    }
```

위에서 설정한 iRule을 해당 Gateway의 SectionName 부분에 적용을 합니다.

```yaml
apiVersion: "gateway.k8s.f5net.com/v1alpha1"
kind: BNKNetPolicy
metadata:
  name: netpolicy-modifier
  namespace: web
spec:
  extensionRefs:
  - kind: F5BigCneIrule
    name: irule-modifier
    group: k8s.f5net.com
  targetRefs:
  - kind: Gateway
    name: modifier-gw
    group: gateway.networking.k8s.io
    sectionName: http-modifier
```
